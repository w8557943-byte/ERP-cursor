# 云函数查询优化指南

## 概述

本文档说明如何优化 `cloudfunctions/erp-api/index.js` 中的数据库查询，主要目标是将查询limit从默认的20-100提升至500，减少往返次数，提升性能。

---

## 优化原则

1. **安全优化**：不修改核心业务逻辑，仅调整查询参数
2. **向后兼容**：保持API接口不变
3. **可回滚**：所有修改都可以快速回滚
4. **性能监控**：配合前端性能监控验证效果

---

## 需要优化的函数

### 1. getOrders (line ~1768)

**当前问题**:
- 默认limit较小（通常20-100）
- 多次往返获取数据
- 未充分利用云开发500条/次的上限

**优化方案**:
```javascript
// 位置: line ~1768
async function getOrders(params = {}) {
  const {
    page = 1,
    limit = 500,  // 从 20 提升至 500
    status,
    customerId,
    dateRange,
    orderType,
    purchaseCategory,
    excludeOrderType,
    keyword
  } = params

  // 计算分页参数
  const safeLimit = Math.min(Math.max(Number(limit) || 500, 1), 500)  // 最大500
  const safePage = Math.max(Number(page) || 1, 1)
  const skip = (safePage - 1) * safeLimit

  // ... 其余代码保持不变
}
```

**预期效果**:
- 单次请求获取500条订单（vs 原来20-100条）
- 减少云函数调用次数 60-80%
- 配合前端缓存，大幅提升列表加载速度

## 实施步骤

### 步骤1: 备份当前云函数
```bash
# 在本地创建备份
cp cloudfunctions/erp-api/index.js cloudfunctions/erp-api/index.js.backup
```

### 步骤2: 修改默认limit值
在每个查询函数中，将 `limit` 参数的默认值从 20/50/100 改为 500

### 步骤3: 部署云函数
```bash
# 使用云开发CLI部署
tcb fn deploy erp-api

# 或使用微信开发者工具上传
```

### 步骤4: 验证效果
1. 打开性能监控面板 (`/performance`)
2. 观察以下指标:
   - 云函数调用量是否减少
   - 缓存命中率是否提升
   - 平均响应时间是否降低
3. 检查数据完整性（确保所有数据都能正确加载）

### 步骤5: 回滚方案（如需要）
```bash
# 恢复备份文件
cp cloudfunctions/erp-api/index.js.backup cloudfunctions/erp-api/index.js

# 重新部署
tcb fn deploy erp-api
```

## 性能预期

### 优化前
- 订单列表加载: 5-10次云函数调用（每次20-100条）
- 总耗时: 2-3秒
- 云函数调用量: 高

### 优化后
- 订单列表加载: 1-2次云函数调用（每次500条）
- 总耗时: 0.5-1秒
- 云函数调用量: 减少60-80%

### 配合缓存后
- 首次加载: 0.5-1秒
- 后续加载: <100ms（缓存命中）
- 云函数调用量: 减少80-90%

## 注意事项

### 1. 数据量监控
- 如果单个集合数据量超过10万条，考虑实施分页策略
- 监控云函数执行时间，确保不超时

### 2. 内存使用
- 500条数据的内存占用通常在1-5MB
- 云函数内存限制为256MB，足够处理

### 3. 网络传输
- 500条数据的JSON大小通常在100-500KB
- 云开发响应大小限制为6MB，远未达到上限

### 4. 索引依赖
- 确保已创建相应的数据库索引
- 否则大limit可能导致全表扫描，反而降低性能

---

### 关键指标
1. **云函数调用量**: 目标减少50%+
2. **平均响应时间**: 目标<500ms
3. **缓存命中率**: 目标>80%
4. **错误率**: 目标<1%

### 监控方式
- 使用性能监控面板实时查看
- 定期导出性能数据分析
- 设置告警阈值
