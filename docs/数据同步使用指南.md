# PC端与小程序云开发数据同步使用指南

## 📋 概述

本指南详细介绍了PC端ERP系统与微信小程序云开发之间的数据同步解决方案，确保两端数据保持一致性和实时性。

## 🏗️ 系统架构

### 核心组件

1. **同步管理器 (SyncManager)**
   - 统一管理所有同步服务
   - 提供完整的同步生命周期管理
   - 支持WebSocket实时通信

2. **增强同步服务 (EnhancedSyncService)**
   - 支持双向数据同步
   - 批量处理数据变更
   - 智能冲突检测与解决

3. **一致性检查服务 (ConsistencyCheckService)**
   - 定期检查数据一致性
   - 自动修复数据差异
   - 生成详细的一致性报告

4. **冲突解决服务 (ConflictResolutionService)**
   - 支持多种冲突解决策略
   - 自动或手动解决冲突
   - 记录冲突解决历史

5. **同步监控服务 (SyncMonitorService)**
   - 实时监控同步状态
   - 性能指标跟踪
   - 异常告警机制

6. **增量同步服务 (IncrementalSyncService)**
   - 高效的增量数据同步
   - 基于时间戳的变更检测
   - 批量处理优化

7. **数据回滚服务 (DataRollbackService)**
   - 自动数据备份
   - 支持多点回滚
   - 快速数据恢复

## 🚀 快速开始

### 1. 安装依赖

```bash
cd d:\ERP-cursor\app\backend
npm install
```

### 2. 配置环境变量

创建 `.env` 文件：

```env

DATABASE_NAME=erp_system

# 微信小程序配置

MINIPROGRAM_APP_SECRET=your_miniprogram_app_secret
WECHAT_CLOUD_ENV_ID=your_cloud_environment_id

# 安全配置

ENCRYPTION_KEY=your_32_character_encryption_key

# 日志配置
LOG_LEVEL=info
```

### 3. 启动服务

```bash

npm run dev

# 启动前端服务
cd ../frontend
npm run dev
```

## 🔧 冲突解决策略

### 支持的策略

1. **时间戳策略 (timestamp)**
   - 以最新的时间戳为准
   - 适用于大多数场景

2. **版本号策略 (version)**
   - 以最高的版本号为准
   - 需要实现版本控制机制

3. **服务器优先 (server_wins)**
   - 总是以服务器数据为准
   - 适用于服务器权威性高的场景

4. **客户端优先 (client_wins)**
   - 总是以客户端数据为准
   - 适用于客户端数据更准确的场景

5. **合并策略 (merge)**
   - 尝试合并冲突的数据
   - 适用于可以合并的场景

6. **手动策略 (manual)**
   - 需要人工介入解决
   - 适用于重要的业务数据

### 配置示例

```javascript
// 设置冲突解决策略
const conflictConfig = {
  strategy: 'timestamp',
  priority: 'server',
  autoResolve: true,
  notification: true
}

// 为特定集合设置策略
const collectionConfig = {
  orders: {
    conflictResolution: 'timestamp'
  },
  customers: {
    conflictResolution: 'server_wins'
  },
  products: {
    conflictResolution: 'merge'
  }
}
```

## 🔄 数据回滚

### 自动备份

系统会自动创建数据备份点：

- **备份频率**: 每小时一次
- **保留时间**: 30天
- **最大备份点**: 50个

### 回滚类型

1. **自动回滚**
   - 同步失败时自动回滚
   - 基于预设条件触发

2. **手动回滚**
   - 管理员手动触发
   - 选择特定备份点

3. **点时间恢复**
   - 恢复到指定时间点
   - 精确到分钟级别

### 回滚API

```javascript
// 执行数据回滚
const rollbackResult = await dataRollbackService.performRollback({
  type: 'manual',
  rollbackPointId: 'rollback_123456',
  reason: '数据同步错误'
})

// 获取可用回滚点
const rollbackPoints = await dataRollbackService.getAvailableRollbackPoints()

// 删除旧回滚点
await dataRollbackService.cleanupOldRollbackPoints()
```

## 🔍 故障排除

### 常见问题

#### 1. 同步失败

**症状**: 同步任务频繁失败

**解决方案**:
```javascript
// 检查同步配置
const config = await syncManager.getSyncConfig()
console.log('当前配置:', config)

// 检查错误日志
const logs = await syncManager.getSyncHistory({ 
  status: 'failed', 
  limit: 10 
})
console.log('最近的失败记录:', logs)
```

#### 2. 数据不一致

**症状**: PC端和小程序端数据不一致

**解决方案**:
```javascript
// 执行一致性检查
const checkResult = await syncManager.performConsistencyCheck({
  autoFix: true,
  detailedReport: true
})

// 查看不一致的数据
if (checkResult.inconsistencies.length > 0) {
  console.log('发现不一致:', checkResult.inconsistencies)
}
```

#### 3. 冲突解决失败

**症状**: 冲突无法自动解决

**解决方案**:
```javascript
// 检查冲突详情
const conflicts = await conflictResolutionService.getUnresolvedConflicts()
console.log('未解决的冲突:', conflicts)

// 手动解决冲突
await conflictResolutionService.resolveConflictManually({
  conflictId: 'conflict_123',
  resolution: 'server_wins',
  reason: '服务器数据更准确'
})
```

#### 4. 性能问题

**症状**: 同步速度慢，响应时间长

**解决方案**:
```javascript
// 检查性能指标
const performance = await syncMonitorService.getPerformanceMetrics()
console.log('性能指标:', performance)

// 调整批量大小
await syncManager.updateConfig({
  performance: {
    batchSize: 500,      // 减小批量大小
    maxConcurrent: 3     // 减少并发数
  }
})
```
