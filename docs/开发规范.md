# 荣禾ERP系统 - 开发规范

## 日志规范

### 使用统一日志工具

**引入日志工具：**
```javascript
const { logger } = require('./utils/logger');
```

**日志级别使用：**

```javascript
// DEBUG - 调试信息（仅开发环境）
logger.debug('ModuleName', '详细的调试信息', { data: someData });

// INFO - 一般信息
logger.info('ModuleName', '操作成功', { userId: 123 });

// WARN - 警告信息
logger.warn('ModuleName', '配置缺失，使用默认值');

// ERROR - 错误信息
logger.error('ModuleName', '操作失败', error);
```

### 禁止使用的日志方式

❌ **禁止直接使用 console.log**
```javascript
// 错误示例
console.log('用户信息:', userInfo);
console.log('密码:', password);
```

✅ **使用 logger 并注意敏感信息**
```javascript
// 正确示例
logger.info('Login', '用户登录成功');
// 敏感信息会自动脱敏
logger.debug('UserData', '用户数据', { password: '123456' }); // 输出: { password: '***' }
```

## 错误处理规范

### 不要吞掉错误

❌ **错误示例**
```javascript
try {
  // some code
} catch (_) {
  // 静默失败，无法追踪问题
}
```

✅ **正确示例**
```javascript
try {
  // some code
} catch (error) {
  logger.error('ModuleName', '操作失败', error);
  // 根据情况决定是否抛出或处理
}
```

## 代码注释规范

### TODO 注释

使用 TODO 时必须包含：
- 负责人
- 预计完成时间
- 简要说明

```javascript
// TODO(@username, 2026-01-20): 实现缓存失效机制
// TODO: 优化查询性能 - 添加索引
```

### 函数注释

```javascript
/**
 * 用户登录
 * @param {string} username - 用户名
 * @param {string} password - 密码
 * @returns {Promise<Object>} 登录结果
 */
async function login(username, password) {
  // ...
}
```

## 安全规范

### 1. 环境变量

- ✅ 使用 `.env` 文件存储敏感配置
- ✅ 确保 `.env` 在 `.gitignore` 中
- ✅ 提供 `.env.example` 模板
- ❌ 不要在代码中硬编码密钥

### 2. 敏感信息

- ❌ 不要在日志中输出密码、token、sessionId
- ❌ 不要在前端代码中暴露 API 密钥
- ✅ 使用 logger 自动脱敏功能

### 3. CORS 配置

- ❌ 生产环境禁止使用 `CORS_ORIGINS=*`
- ✅ 限制为特定域名列表

## Git 提交规范

### 提交信息格式

```
<type>(<scope>): <subject>

<body>

<footer>
```

**Type 类型：**
- `feat`: 新功能
- `fix`: 修复 bug
- `docs`: 文档更新
- `style`: 代码格式化
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具变动
- `security`: 安全修复

**示例：**
```
feat(auth): 添加双因素认证

- 实现 TOTP 验证
- 添加备用验证码
- 更新用户设置页面

Closes #123
```

## 测试规范

### 单元测试

- 关键业务逻辑必须有测试覆盖
- 测试文件命名：`*.test.js` 或 `*.spec.js`
- 测试覆盖率目标：> 80%

### 集成测试

- API 接口必须有集成测试
- 使用真实数据库进行测试
- 测试后清理数据

## 性能规范

### 1. 避免过度缓存

```javascript
// ❌ 缓存时间过长
const CACHE_TTL = 3600000; // 1小时

// ✅ 根据数据特性设置合理的缓存时间
const CACHE_TTL = {
  userInfo: 300000,    // 5分钟
  orderList: 60000,    // 1分钟
  staticData: 3600000  // 1小时
};
```

### 2. 数据库查询优化

- 使用索引
- 避免 N+1 查询
- 使用分页
- 限制返回字段
